▌ MISSION-CONTROL
LAN-shared Claude Code analytics — setup & run guide

MISSION-CONTROL aggregates Claude Code usage from every device on your local network into one persistent dashboard. One host machine runs the backend + PostgreSQL; every device (including the host) runs a lightweight collector that pushes its local session data to the host. Any device on the LAN can then view the combined dashboard after logging in. Nothing leaves your network.

Dashboard URL (open on any device on this network)
http://<host-ip>:9000

Log in with the shared DASHBOARD_PASSWORD set on the host.

AOn the host device (runs the backend)

Do this once, on the single machine that will store everyone's data.

  1. Install dependencies
    npm install
  2. Create the secrets file and set a dashboard password
    cp .env.example .env

    Edit .env and set DASHBOARD_PASSWORD (and DATABASE_URL if you changed the Postgres defaults).

  3. Start PostgreSQL and apply the schema
    docker compose up -d
npm run db:migrate
  4. Register the host as a device (prints an API key once)
    npm run register-device -- --id host --name "Host" --host

    Copy the printed key into collector.config.json (see collector.config.example.json).

  5. Start everything (Postgres → migrate → backend → host collector)
    ./start.sh

    Or use the missioncontrol shell alias if you set one up. The backend listens on 0.0.0.0:9000 (reachable across your LAN).

BOn every other device (laptop, desktop, …)

Repeat for each additional machine whose Claude Code usage you want to track.

  1. Clone the repo and install
    git clone <repo-url> mission_control
cd mission_control
npm install
  2. Register this device — run this command on the host
    npm run register-device -- --id macbook-air --name "MacBook Air"

    Pick a unique --id per device. The command prints the device key — you'll paste it in the next step.

  3. Create this device's collector config
    cp collector.config.example.json collector.config.json

    Set backendUrl to the host URL above, plus this device's deviceId, deviceName, and the deviceKey from step 2.

  4. Start the collector
    npm run collector

    Leave it running. It pushes new/changed sessions automatically (and a toolkit snapshot on startup + every 5 min).

CView the dashboard (any device)

Open the dashboard URL in a browser on any device on this network and log in with the shared DASHBOARD_PASSWORD:

Dashboard
http://<host-ip>:9000

Use the top-bar device filter (All / per-device) to slice sessions, cost, and the Toolkit view by machine.

DOptional: encrypt the connection (untrusted networks)

By default the dashboard is plain HTTP — fine for a trusted home/office LAN. On an untrusted network, enable self-signed TLS so the password and cookie aren't sent in the clear:

  1. Generate a self-signed certificate (on the host)
    npm run gen-cert
  2. Enable it in .env and restart
    TLS_CERT_FILE=./certs/cert.pem
TLS_KEY_FILE=./certs/key.pem

    The dashboard then serves https://<host-ip>:9000. Browsers show a one-time self-signed warning. For production-grade TLS, front with nginx/Caddy or use Tailscale.
